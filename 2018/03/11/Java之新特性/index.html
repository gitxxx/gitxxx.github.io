Java8

Java 8 (jdk 1.8) 是Java的一个主要版本。于2014年3月18日发布，核心新feature是支持函数式编程。
主要新特性如下：

  1. Lambda表达式 Lambda表达式可做为方法的参数。Lambda表达式可以更简洁地表达单方法类的实例。
  2. Method引用 方法引用提供了非常有用的语法，可以直接引用已有Java类或对象（实例）的方法或构造器。与lambda联合使用，方法引用可以使语言的构造更紧凑简洁，减少冗余代码。
  3. Stream API 新添加的Stream API（java.util.stream） 把真正的函数式编程风格引入到Java中。
  4. Optional类 Optional 类已经成为 Java 8 类库的一部分，用来解决空指针异常。
  5. Default方法 默认方法就是一个在接口里面有了一个实现的方法。
  6. Nashorn javascript引擎 Java 8提供了一个新的Nashorn javascript引擎，它允许我们在JVM上运行特定的javascript应用。
  7. Date Time API 加强对日期与时间的处理。
  8. IO和NIO SelectorProvider基于Solaris事件端口机制的Solaris 新实现。性能改进String(byte[], *)构造函数和getBytes()方法。

忘记在哪里看到非常好的一句话：小处使用函数式编程，大处使用面向对象编程。使用函数式编程确实可以大大简化编程过程。下面对常使用到的新特性进行说明

Lambda表达式

Lambda表达式是一个匿名函数，即没有函数名的函数。
Java 8 lambda表达式的语法：
(params) -> expression
(params) -> statement
(params) -> { statements }

lambda用途：

  • lambda表达式替换匿名类
  • lambda表达式对集合类进行迭代

Method引用

在Java8中Method也是对象，可通过名字引用，Method引用是为了Lambda简写而设计的。
使用Lambda表达式，如果Lambda表达式仅仅调用一个已存在的方法，不做其他事情，通过方法名称引用更方便。
适合：调用一个已存在的方法
不适合：往引用的方法传其它参数
Method调用格式：”类型::方法名”

Method引用分为：

  • 引用静态方法
  • 引用实例对象方法
  • 引用构造函数

Stream API

Stream 不是集合元素，它不是数据结构并不保存数据，它是有关算法和计算的，它更像一个高级版本的Iterator。
集合类新增的stream()方法用于把一个集合变成Stream，然后，通过filter()、map()等实现Stream的变换。Stream还有一个forEach()来完成每个元素的迭代。

Optional类

OPtional是一个容器对象，Optional是为了解决null问题的类，Optional提供三个静态方法构造Optional对象：

  • Optional.of(T value) 通过传入参数不能为null，通过value来构造一个Optional，返回的Optional包含了value这个值。
  • Optional.ofNullable(T value) 传入的参数可以为null。该方法会判断传入的参数是否为 null，如果为 null 的话，返回的就是 Optional.empty()。
  • Optional.empty() 该方法用来构造一个空的Optional，即该Optional中不包含值

其它方法：

  • isPresent() 如果存在值，isPresent()将返回true并可以使用get()得到Optional内的value。
  • orElse() 值不存在，返回一个指定的默认值。

Java9

于2017年9月21日发布。主要新特性如下：

  1. Java平台级模块系统
  2. Linking
  3. JShell : 交互式 Java REPL
  4. 改进的 Javadoc
  5. 集合工厂方法
  6. 改进的 Stream API
  7. 私有接口方法
  8. HTTP/2
  9. 多版本兼容 JAR
