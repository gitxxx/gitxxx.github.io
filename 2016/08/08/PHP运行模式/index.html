简介

生产环境中有大量使用PHP开发的动态网站，PHP网站开发完成后，需要部署到web服务器然后运行，与PHP搭配的web服务器有如下三种：

  • Apache
  • Nginx
  • ligthttpd

Apache中运行模式

  • CGI模式；
  • FastCGI模式；
  • Apache模块DLL；

下面分别对这三种运行模式进行说明。

CGI模式

CGI英文全称是 Common Gateway Interface，通常翻译为共同网关接口，是HTTP服务器与机器上的其他程序进行通信的一个接口。这个“其他程序”可以使用任何计算机语言来编写，它通过CGI这个接口从HTTP服务器取得输入，然后把运行的结果又通过CGI这个接口交给HTTP服务器，而HTTP服务器把这个结果送给浏览器。

CGI的出现让WEB从静态变为为动态，随着Web的越来越普及，很多的网站的都需要有动态的页面，以便与浏览者互交。CGI方式的缺点也越来越突出。因为HTTP要生成一个动态页面，系统就必须启动一个新的进程以运行CGI程序，不断地fork是一项很消耗时间和资源的工作。这就出现了FastCGI。

FastCGI模式

FastCGI是语言无关的、可伸缩架构的CGI开放扩展，其主要行 为是将CGI解释器进程保持在内存中并因此获得较高的性能。

Apache模块DLL

PHP是与Web服务器一起启动并运行的。以apache模块方式运行PHP。

Nginx和ligthttpd运行模式

Nginx和ligthttpd中常使用FAstCGI的运行模式，如下两种组合：

  • ligthttpd + spawn-fcgi
  • nginx + PHP-FPM(也可使用spawn-fcgi)

流行组合

目前在HTTPServer这块基本可以看到有三种stack比较流行：

  1. Apache+mod_php5
  2. lighttp+spawn-fcgi
  3. nginx+PHP-FPM

三者中后两者性能稍优，但是Apache由于有丰富的模块和功能，目前来说仍旧是老大。有人测试nginx+PHP-FPM在高并发情况下可能会达到Apache+mod_php5的5~10倍，现在nginx+PHP-FPM使用的人越来越多。
