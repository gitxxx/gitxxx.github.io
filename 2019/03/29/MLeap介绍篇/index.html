MLeap是什么？

MLeap是machine learning pipelines的常用序列化格式和执行引擎。它支持Spark，Scikit-learn和Tensorflow，用于training pipelines并将它们导出到MLeap Bundle。Serialized pipelines (bundles)可以反序列化回Spark，Scikit-learn，TensorFlow graphs或MLeap pipeline，以用于评分引擎（API服务器）。

为什么是MLeap？

许多使用Spark和Scikit学习的公司很难将他们的研究ML /数据管道模型部署到生产API服务中。如果公司不希望在他们的API堆栈中使用Python或不使用Google ML Cloud，即使使用Tensorflow也很难设置这些服务。MLeap提供简单的界面来执行整个ML流水线，从特征变换器到分类器，回归，聚类算法和神经网络。

Portable model

models仍是自己的models。使用MLeap Bundles随身携带它们。像Microsoft Azure和Google ML这样的平台可以将您锁定在他们的服务包中。无论您走到哪里，MLeap都可以让您随身携带model。

Spark，Scikit-learn和Tensorflow：One Runtime

结合和匹配ML技术成为一项简单的任务。与其要求整个开发团队准备研究管道生产，不如将其导出到MLeap Bundle并在需要的地方运行pipeline。统一运行时的其他好处：

  • 使用Spark，Scikit-learn或Tensorflow训练不同的pipeline，然后将它们导出到一个MLeap Bundle 文件，可将其部署到任意位置
  • 如果您使用Scikit进行研发，但Spark提供了更好的算法，则可以将Scikit ML管道导出到Spark，在Spark中训练新model，然后使用MLeap runtime部署到生产环境

常见的序列化

除了提供有用的执行引擎之外，MLeap Bundles还为ML特性提取器和算法提供了通用序列化格式，这些特征提取器和算法可以通过Spark，Scikit-learn，Tensorflow和MLeap进行导出和导入。这意味着您可以轻松地在这些技术之间转换pipelines，具体取决于您需要执行管道的位置。

无缝集成

在大多数情况下，在Spark或Scikit-learn中我们不修改任何内部代码或要求继承transformers 。对于Tensorflow，我们尽可能多地使用内建操作，在MLeap不存在时才实施自定义操作。这意味着对现有管道的代码更改对于启动和运行MLeap来说是最小的。对于许多用例，不需要进行任何更改，您可以简单地导出到MLeap Bundle或将其部署到Combust API server以开始立即使用pipeline。
