ES集群概念

  • Node 一个运行中的 Elasticsearch 实例。
  • Cluster 集群是由一个或者多个拥有相同 cluster.name 配置的节点组成，它们共同承担数据和负载的压力。当有节点加入集群中或者从集群中移除节点时，集群将会重新平均分布所有的数据。
  • Master 管理Node、index级别的增删改、不负责document的事情。当一个Node被选举成为master节点时， 它将负责管理集群范围内的所有变更，例如增加、删除索引，或者增加、删除节点等。 而主节点并不需要涉及到文档级别的变更和搜索等操作，所以当集群只拥有一个master节点的情况下，即使流量的增加它也不会成为瓶颈。 任何Node都可以成为主节点。我们的示例集群就只有一个节点，所以它同时也成为了主节点。
  • Coordinating Node 处理读取请求的节点，协调结点在每次请求的时候都会通过轮询所有的副本分片来达到负载均衡。所有节点都可以是协调结点。

作为用户，我们可以将请求发送到Cluster中的任何Node ，包括Master节点。 每个Node都知道任意文档所处的位置，并且能够将我们的请求直接转发到存储我们所需文档的Node。 无论我们将请求发送到哪个节点，它都能负责从各个包含我们所需文档的节点收集回数据，并将最终结果返回給客户端。 Elasticsearch对这一切的管理都是透明的。

Index逻辑概念

  • Index 相当于关系数据库中的database概念，一个集群中可以包含多个索引。这个是个逻辑概念。
  • type 每个Index可以从逻辑上划分成几个type。
  • documents 一条需要被索引的记录，包含多个fields。
  • Field documents被划分为多个字段，每个字段可以有自己的字段类型。

Index知识点：

  • mapping：相当于数据库中的schema，用来约束字段的类型，不过 Elasticsearch 的 mapping 可以自动根据数据创建。
  • settings：关于index的设置信息。
  • templates：用于复用mapping。设定一个模板后，当创建index时，会根据模板，生成相应的mapping。

Index物理概念

  • primary shard Index的子集，Index可以切分成多个分片，分布到不同的集群节点上。shard对应的是 Lucene 中的索引。
  • replica shard 每个primary shard的副本，可以没有副本，但是为了数据可靠性，工业界不会存在没有replica的情况，replica可以有一个或者多个。
  • Allocation 分配，将shard分配给某个Node的过程，包括分配primary shard或者replica shard。如果是副本，还包含从主分片复制数据的过程。
  • segment 每个shard包含多个segment，每一个segment都是一个倒排索引；在查询的时，会把所有的segment查询结果汇总归并后做为最终的shard查询结果返回；生成segment的数据来源，是内存中的buffer。
  • commit文件 记录索引内的所有segment，
  • translog 当一个文档被索引，它被加入到内存buffer，同时加到translog日志中。
